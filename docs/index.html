How it works

01
Drop a rough requirement. Tony Stark (PM) asks a few sharpening questions, then commissions research and writes the full spec.
02
Review gates, not code. At each of 5 gates you get an HTML summary and choose to approve or push back. No code is written until you sign off.
03
Agents stay in their lane. Each one has a strict role and passes work through shared files. Sessions can be resumed at any gate.
04
The pipeline gets smarter. After every run, Yoda writes learnings that every agent reads at startup — improving with each build.

The pipeline

G1
Spec Alignment
Clarify requirements, run competitive research, write the brief with success metrics, validate assumptions.
Tony Stark · Sherlock Holmes · Hermione
G2
Design Options
3–5 UX approaches built as live routes, each with performance notes and research annotations. You pick one; the rest are torn down.
Jonathan Ive · Jackie Chan · Bob · Sherlock
G3
Technical Contract
Full API contract — endpoints, shapes, error handling, auth, loading strategy, data model — reviewed by all consumers.
Bob the Builder · Jackie Chan · Jonathan Ive
G3½
Tech Spec Review
Schema DDL, exact files touched, build sequence, rollback plan. Schema changes to existing tables need explicit approval.
Bob the Builder · Jackie Chan · Hermione
G4
QA + Sign-off
Test plan, execution results, design system compliance, coverage gaps, go/no-go recommendation. Ship or iterate.
Gordon Ramsay · Sherlock · Ive · Jackie · Bob

The agents

Tony Stark
PM & Orchestrator
Owns the what and why. Drives the pipeline, resolves conflicts, writes the brief, maintains the decision log.
Sherlock Holmes
Research & Analysis
Competitive analysis, assumption audits, mental model maps. Silent red-flag detector in later gates.
Jonathan Ive
UX Design
Produces 3–5 interaction specs with aesthetic direction. Reviews contract for design intent. Validates final output.
Bob the Builder
API & Architecture
Designs API contracts, schema DDL, build sequence, and rollback plans. Nothing ships without Bob's sign-off.
Jackie Chan
Frontend Engineer
Builds all design options as live routes. Implements the chosen option to production quality.
Gordon Ramsay
QA & Testing
Writes and runs the test plan, produces gap reports, issues the final go/no-go recommendation.
Hermione Granger
Project Management
Readiness checks at every gate. Surfaces blockers and owns state.json accuracy throughout.
Yoda
Learning & Memory
After every run, distils outcomes into learnings.md — read by all agents at startup to improve the next build.

Install

Requires Claude Code. Run from your application project, not this repo.

# Add the marketplace
/ plugin marketplace add D4rsh4k/autonomous-workflow
# Install the plugin
/ plugin install autonomous-workflow@d4rsh4k-plugins
# Activate and run
/ reload-plugins
/ autonomous-workflow:build <your requirement>
Seeing "Unknown skill: plugin" on Desktop? Update Claude Code Desktop to the latest version and restart. Or use the terminal: npm install -g @anthropic-ai/claude-code
