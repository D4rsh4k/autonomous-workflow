How it works

Brief once.
Ship with confidence.

You stay in control without being in the weeds. The agents do the work; you make the calls.

01
Drop a rough requirement
2–3 sentences is enough. Tony Stark (your PM) asks sharpening questions, commissions research, then writes the full spec with success metrics.
02
Review gates, not pull requests
At each of the 5 gates you approve or push back. A browser-ready HTML summary is generated every time. No code is written until you sign off.
03
Each agent owns its domain
Agents have strict roles, never improvise outside them, and pass work through shared artifacts. Your session can resume from any gate.
04
The pipeline learns
After every run, Yoda distils outcomes into learnings.md. Every subsequent run reads those learnings and improves on them.

The crew

Eight agents.
One shared goal.

Each operates only within its domain, reads all prior artifacts, and applies learnings from every previous run.

TS
Tony Stark
PM & Orchestrator
Owns the what and why. Drives the pipeline forward, resolves agent conflicts, writes the brief, and maintains the decision log.
SH
Sherlock Holmes
Research & Analysis
Competitive analysis, assumption audits, mental model maps. Silent red-flag detector in later gates — speaks only when he sees a friction pattern.
JI
Jonathan Ive
UX Design
Produces 3–5 interaction specs with aesthetic direction and rationale. Reviews the contract for design intent. Validates final output for experience fidelity.
BB
Bob the Builder
API & Architecture
Designs the full API contract, schema DDL, build sequence, and rollback plan. Nothing gets built without Bob's sign-off.
JC
Jackie Chan
Frontend Engineer
Builds all design options as live routes; implements the chosen option to production quality. Never creates components without written justification.
GR
Gordon Ramsay
QA & Testing
Writes and executes the test plan, produces gap reports and component audits, issues the final go/no-go recommendation. No tolerance for sloppy work.
HG
Hermione Granger

Readiness checks at every gate. Tracks deliverable checklists, surfaces blockers immediately, and owns the accuracy of state.json.
YO
Yoda

After every run, analyses decisions and outcomes and writes concise learnings that all agents read at startup — making every subsequent run better.
Get started

Install in
two commands.

  1. 1 Open Claude Code (terminal or Desktop, latest version) in your application project.
  2. 2 Add the marketplace and install the plugin with the two commands opposite.
  3. 3 Run /reload-plugins, then start your first build.
Installation
# Step 1 — add the marketplace catalog
/
plugin marketplace add D4rsh4k/autonomous-workflow
# Step 2 — install the plugin
/
plugin install autonomous-workflow@d4rsh4k-plugins
# Step 3 — activate
/
reload-plugins
# Step 4 — start your first build
/
autonomous-workflow:build <your requirement>

Desktop showing "Unknown skill: plugin"? Update Claude Code Desktop to the latest version and restart. Or use the terminal: npm install -g @anthropic-ai/claude-code then run claude from your project directory.
