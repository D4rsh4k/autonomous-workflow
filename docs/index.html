How it works

Brief once.
Ship with confidence.

You stay in control without being in the weeds. The agents do the work; you make the calls.

✦
Drop a rough requirement
2–3 sentences is enough. Tony Stark (your PM) asks sharpening questions, then commissions research and writes the spec.
◈
Review gates, not pull requests
At each of the 5 gates you see a styled HTML summary and approve or push back. No code is written until you sign off on the architecture.
◎
Agents stay in their lane
Each agent has a strict role, never improvises outside it, and passes work to the next agent through shared artifacts your session can resume at any time.
⟳
The pipeline gets smarter
After every run, Yoda distils what worked into learnings.md. Every subsequent run reads those learnings and improves.
The pipeline

Five gates.
Zero surprises.

Every gate ends with a human decision point and a browser-ready HTML summary of all artifacts.

The crew

Eight agents.
One shared goal.

Each agent operates only within its domain, reads all prior artifacts, and applies learnings from every previous run.

TS
Tony Stark
PM & Orchestrator
Owns the what and why. Drives the pipeline, resolves agent conflicts, writes the brief, and maintains the decision log.
SH
Sherlock Holmes
Research & Analysis
Competitive analysis, assumption audits, mental model maps. Silent red-flag detector in later gates.
JI
Jonathan Ive
UX Design
Produces 3–5 interaction specs with aesthetic direction. Reviews contract for design intent. Validates final output for experience fidelity.
BB
Bob the Builder
API & Architecture
Designs the full API contract, schema DDL, build sequence, and rollback plan. Nothing gets built without Bob signing off.
JC
Jackie Chan
Frontend Engineer
Builds all design options as live routes. Implements the chosen option to production quality. Never creates components without justification.
GR
Gordon Ramsay
QA & Testing
Writes and executes the test plan, produces gap reports and component audits, issues the final go/no-go recommendation.
HG
Hermione Granger
Project Management
Readiness checks at every gate. Tracks deliverable checklists, surfaces blockers, owns state.json accuracy.
YO
Yoda
Learning & Memory
After every run, analyses decisions and outcomes and writes concise learnings that every agent reads at startup — making the next run better.
Get started

Install in
two commands.

Requires Claude Code CLI. Run from your application project directory — not this repo.

claude — zsh
# Add the marketplace catalog
/
plugin marketplace add D4rsh4k/autonomous-workflow
# Install the plugin
/
plugin install autonomous-workflow@d4rsh4k-plugins
# Activate + run your first build
/
reload-plugins
# Start a pipeline with a rough requirement
/
autonomous-workflow:build <your requirement>
Using Claude Code Desktop? If /plugin shows "Unknown skill", update Desktop to the latest version and restart. Alternatively, install the CLI: npm install -g @anthropic-ai/claude-code and use the terminal.
